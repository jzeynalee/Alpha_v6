## Technical Fix Applied

Before the cycle, the effect‑summary storage used plain symbols as keys,
leading to overwriting when multiple timeframes shared the same symbol.
This has been fixed:

- Effect summaries are now stored with composite keys
  (`BTCUSDT_15m`, `BTCUSDT_1h`, …) in `mech.effect_summary`.
- The built‑in M001 entry (`BTCUSDT` → 15m) remains intact; new entries
  (`BTCUSDT_1h`) are stored separately.
